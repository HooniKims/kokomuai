# 꼬꼬무AI 디자인 시스템

이 문서는 현재 `꼬꼬무AI` 화면에 적용된 디자인을 다른 프로젝트나 화면에도 재사용할 수 있도록 정리한 기준이다. 실제 앱의 런타임 스타일은 `src/presentation/styles.css` 안에서 완결되며, 이 문서는 구현 기준을 설명하는 참고 문서로 사용한다.

## 1. 디자인 방향

`꼬꼬무AI`는 일반 AI 도구처럼 차갑고 기술적인 느낌을 주기보다, 선생님이 수업 안에서 열어 주는 질문형 학습 공간처럼 보여야 한다.

핵심 인상:

- 따뜻한 크림색 종이
- 진한 초록 잉크
- 형광펜으로 표시한 듯한 노란 강조
- 수업 자료와 sticky-note를 떠올리게 하는 추천 카드
- 넓고 읽기 쉬운 학생 채팅 화면
- 조밀하지만 정리된 교사용 설정 화면

## 2. 타이포그래피

기본 폰트는 Paperlogy를 사용한다.

```css
@font-face {
  font-family: "Paperlogy";
  font-weight: 400;
  src: url("/fonts/Paperlogy-4Regular.woff2") format("woff2");
}

@font-face {
  font-family: "Paperlogy";
  font-weight: 700;
  src: url("/fonts/Paperlogy-7Bold.woff2") format("woff2");
}
```

전역 기준:

- 기본 글자 크기: `17px`
- 글자 간격: `-0.02em`
- 줄간격: `1.35`
- 한글 줄바꿈: `word-break: keep-all`
- 긴 단어 보완: `overflow-wrap: break-word`
- 제목 줄바꿈: 가능한 경우 `text-wrap: balance`

텍스트 원칙:

- 학생 화면 문장은 짧고 부드럽게 쓴다.
- “교사가”보다 “선생님이”처럼 실제 수업 맥락에 가까운 표현을 우선한다.
- 버튼과 안내 문구는 기능 설명보다 행동 결과를 바로 알 수 있게 쓴다.
- 개발자용 파일명, 내부 원문명, 경로는 사용자 화면에 노출하지 않는다.

## 3. 색상 토큰

현재 구현 기준 색상:

| 이름 | 값 | 용도 |
|---|---:|---|
| Forest ink | `#1a3300` | 본문, 제목, 주요 테두리 |
| Highlighter yellow | `#ffe95c` | 강조, hover, 선택 상태 |
| Cream paper | `#fcfaf5` | 전체 배경과 패널 바탕 |
| Pencil gray | `#b6b6b6` | 보조 테두리 |
| Whisper gray | `#f1f1f1` | 약한 배경 |
| Sticky-note teal | `#a8e5e5` | 추천 카드 변형 |
| Sticky-note mint | `#d5f5c2` | 추천 카드 변형 |
| Sticky-note blush | `#f6d0ff` | 추천 카드 변형 |
| Terracotta | `#cb5521` | 위험, 삭제, 주의 |
| Soft orange | `#ffd4a8` | 보조 강조 |
| Paper line | `rgb(26 51 0 / 22%)` | 은은한 선 |
| Paper fill | `rgb(252 250 245 / 82%)` | 반투명 패널 |

주의:

- 보라/파랑 그라디언트가 화면 대부분을 차지하는 일반 AI 서비스 느낌은 피한다.
- 크림, 초록, 노랑을 중심으로 쓰되 한 가지 색 계열만 반복하지 않는다.
- 삭제와 위험 동작은 terracotta 계열로 분리해 즉시 구분되게 한다.

## 4. 형태와 표면

기본 형태:

- 패널, 카드: `border-radius: 12px`
- 상단 내비게이션: `border-radius: 16px`
- 버튼: `border-radius: 6px`
- 테두리: `1px solid #1a3300` 또는 보조 회색 선
- 그림자: 강한 부유감보다 얕은 종이 그림자 사용

패널 원칙:

- 페이지 섹션 자체를 카드처럼 과하게 띄우지 않는다.
- 카드 안에 또 카드를 넣는 중첩 카드 구조를 피한다.
- 교사용 화면은 정보 밀도를 유지하고, 학생 화면은 대화 여백을 넓힌다.

## 5. 배경과 히어로

전체 배경은 크림색 종이에 가까운 바탕을 사용하고, 작은 점 패턴으로 학습 노트 느낌을 만든다.

히어로 영역:

- 태블릿에서도 채팅 영역이 빨리 보이도록 과하게 높이지 않는다.
- 제목은 `꼬리에 꼬리를 무는 AI`를 크게 보여준다.
- 소개 문구는 `질문과 대화를 통해 스스로 알아가는, 여러분을 위한 공간입니다.`를 사용한다.
- 핵심 단어에는 형광펜 노란색 강조를 사용할 수 있다.

권장 히어로 크기:

```css
.hero-band {
  min-height: clamp(230px, 28vh, 330px);
}

.hero-copy h1 {
  font-size: clamp(42px, 7vw, 72px);
  line-height: 1.06;
  font-weight: 700;
}
```

## 6. 내비게이션

상단 내비게이션은 얇은 테두리와 둥근 모서리를 가진다.

원칙:

- 높이: 약 `52px`
- 최대 폭: 앱 콘텐츠 폭과 맞춘다.
- 모서리: `16px`
- 배경: 반투명 크림색
- 현재 탭은 노란 강조 또는 진한 초록 채움으로 구분한다.

학생 공유 링크 화면에서는 교사와 관리자 탭을 보여주지 않는다.

## 7. 버튼

기본 버튼은 약간 둥근 사각형이다. 이름이 pill이어도 실제 형태는 과한 캡슐형보다 단정한 6px 라운딩을 우선한다.

상호작용:

- hover: 살짝 위로 이동하고 노란 강조 또는 얕은 그림자를 준다.
- active: 눌린 느낌이 나도록 아래로 1px 이동한다.
- focus-visible: 키보드 접근성을 위해 노란 outline을 표시한다.

버튼 종류:

- 기본: 투명 배경, 초록 테두리
- 주요: 초록 배경, 크림색 텍스트
- 위험: terracotta 계열
- 전송: 44px 정사각형, 명확한 아이콘 또는 짧은 텍스트

## 8. 교사용 폼

교사용 생성 폼은 빈 화면처럼 보이지 않도록 placeholder를 활용한다. placeholder는 희미하게 보이는 예시일 뿐 실제 저장값이 아니다.

폼 원칙:

- 라벨은 작지만 읽기 쉽게 굵게 표시한다.
- 기본 폰트 크기를 지나치게 줄이지 않는다.
- textarea의 브라우저 기본 resize 핸들은 숨긴다.
- 예시 주제는 현재 시연 기준으로 중학교 국어 품사 맥락을 사용한다.

예시 placeholder:

- `품사의 종류와 특성 이해`
- `학생이 문장에서 품사의 쓰임을 살펴보고, 관형사와 부사의 차이를 스스로 설명하도록 돕는다.`

## 9. 성취기준 추천 카드

추천 카드는 선생님이 교육과정을 확인하고 선택하는 보조 영역이다.

구성:

- 안내 문구: `관련된 성취기준이 있으면 하나 이상 선택해 주세요.`
- 기본 카드: 3개
- 추가 후보: `더 보기`로 펼침
- 선택: 다중 선택 가능
- 선택하지 않음: 가장 관련성 높은 1개 자동 반영

카드에 반드시 표시할 정보:

- 성취기준 번호
- 관련성 `상`, `중`, `하`
- 과목
- 영역
- 성취기준 요약
- 추천 이유

표시하지 않을 정보:

- 내부 Markdown 파일명
- 개발자용 원문명
- 로컬 경로

선택 버튼은 카드 맨 윗줄 우측에 둔다. 성취기준 번호와 관련성 표시를 가리거나 줄바꿈을 어색하게 만들면 안 된다.

## 10. 학생 채팅 화면

학생 화면의 우선순위는 넓고 편안한 대화 경험이다.

PC 기준:

- 일반적인 AI 채팅 서비스처럼 넓은 대화 영역을 제공한다.
- 정보 패널은 좁게, 채팅 패널은 넓게 둔다.
- 채팅 메시지 최대 폭은 너무 좁지 않게 설정한다.

태블릿과 모바일:

- 채팅 패널을 먼저 보여준다.
- 정보 패널은 아래로 이동할 수 있다.
- 입력창은 항상 접근 가능해야 한다.
- 응답 생성 중에도 채팅창이 화면 아래로 밀려 숨지 않게 한다.

채팅 동작:

- 새 메시지가 생기면 메시지 목록 내부가 자연스럽게 아래로 스크롤된다.
- 페이지 전체가 불필요하게 튀어 이동하지 않게 한다.
- AI 답변의 Markdown 문법은 실제 서식으로 렌더링한다.
- 시작 멘트는 학생이 첫 질문을 쉽게 쓰도록 돕는 안내여야 한다.

## 11. 공유와 삭제 피드백

공유 링크 복사:

- 화면 상단 고정 알림으로 시선을 빼앗지 않는다.
- 해당 챗봇 항목 아래나 버튼 근처에 잠깐 표시한다.
- 일정 시간이 지나면 자동으로 사라진다.

삭제 확인:

- 삭제 버튼을 누르는 즉시 삭제하지 않는다.
- 삭제 버튼 근처에 확인 메시지를 보여준다.
- 확인 후 삭제되면 목록에서 바로 사라진다.
- 여러 챗봇 삭제는 선택 후 한 번 더 확인한다.

## 12. 반응형 기준

권장 기준:

- `1280px` 이상: 학생 화면 최대 폭을 넓히고 채팅 높이를 충분히 확보한다.
- `1180px` 이하: 정보 패널 폭을 줄이고 채팅 영역을 유지한다.
- `920px` 이하: 채팅을 먼저, 정보 패널을 아래로 재배치한다.
- `560px` 이하: 화면 폭을 거의 모두 사용하고 버튼 터치 영역을 유지한다.

핵심 기능은 화면 크기가 줄어도 숨기지 않는다. 배치만 바꾼다.

## 13. 재사용 체크리스트

다른 화면에 이 디자인을 적용할 때 확인한다.

- Paperlogy 폰트를 적용했는가?
- 전역 `letter-spacing: -0.02em`, `line-height: 1.35`를 유지했는가?
- 한글 줄바꿈에 `word-break: keep-all`을 적용했는가?
- 크림 종이, 초록 잉크, 노란 형광펜의 관계가 유지되는가?
- 버튼 hover와 active 피드백이 있는가?
- 삭제, 복사, 선택 같은 상태 변화가 누른 위치 근처에서 보이는가?
- 학생 화면에서 교사/관리자 요소가 보이지 않는가?
- 추천 카드에서 성취기준 번호와 관련성 등급이 보이는가?
- 개발자용 파일명이나 내부 경로가 사용자 화면에 보이지 않는가?
- 태블릿에서 히어로가 채팅 영역을 과하게 밀어내지 않는가?
