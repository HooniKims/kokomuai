# 발표 정리: 꼬꼬무AI는 무엇이 다른가

## 1. 한 문장 소개

꼬꼬무AI는 학생에게 정답을 바로 주는 챗봇이 아니라, 교사가 설정한 수업 범위와 2022 개정 교육과정 안에서 학생이 스스로 생각을 정리하도록 돕는 질문형 학습 챗봇입니다.

교사는 수업 주제, 학습 목표, 학교급, 학년군, 과목, 성취기준, 힌트 강도, 페르소나를 설정하고 학생에게 공유 링크를 제공합니다. 학생은 별도 가입 없이 공유 링크로 들어와 질문하고, 자신의 대화 기록을 PDF 또는 TXT로 내려받을 수 있습니다.

## 2. 발표 핵심 메시지

대부분의 AI 챗봇은 질문을 받으면 빠르게 답을 줍니다. 꼬꼬무AI는 반대로 설계했습니다.

학생이 질문하면 곧바로 정답을 말하지 않고, 학생의 생각을 조금씩 좁혀 가는 질문을 이어 갑니다. 서비스의 목표는 더 빠른 정답 생성이 아니라 더 깊은 이해입니다.

교사가 만든 수업의 울타리 안에서 AI가 질문, 힌트, 비유, 점검 질문을 제공하고, 학생은 자신의 생각을 다시 말하고 정리하며 개념에 가까워집니다.

## 3. 문제의식

생성형 AI가 학교 현장에 들어오면 다음 문제가 먼저 생깁니다.

- 학생이 스스로 생각하지 않고 답만 가져갈 수 있다.
- 실제 답안 생성 도구로 쓰일 수 있다.
- 교사가 정한 수업 범위를 벗어난 대화가 계속될 수 있다.
- 학생 개인정보가 저장되거나 외부로 전송될 수 있다.
- 교육과정과 무관한 일반 챗봇이 수업을 흔들 수 있다.

꼬꼬무AI는 이 문제를 기능의 중심에 두고 설계했습니다. AI를 정답 생성기가 아니라 교사가 설계한 수업 안에서 학생의 사고를 이끄는 도구로 제한합니다.

## 4. 일반 챗봇과의 차이

| 구분 | 일반 AI 챗봇 | 꼬꼬무AI |
|---|---|---|
| 목표 | 질문에 대한 답 제공 | 학생 사고 정리와 개념 이해 |
| 응답 방식 | 정답, 설명, 요약을 바로 제공 | 질문, 힌트, 비유, 점검 질문으로 유도 |
| 수업 범위 | 사용자가 묻는 대로 넓게 확장 | 교사가 정한 주제와 교육과정 범위 안에서 응답 |
| 교육과정 | 별도 기준 없이 일반 지식 사용 | 2022 개정 교육과정과 성취기준을 기준으로 사용 |
| 성취기준 | 사용자가 직접 찾아 넣어야 함 | 관련 성취기준을 추천하고 여러 개 선택 가능 |
| 학생 입장 | 일반 서비스 화면에서 시작 | 공유 링크로 학생 화면만 진입 |
| 학생 기록 | 서버에 저장될 수 있음 | 기본적으로 학생 기기 브라우저에만 저장 |
| 개인정보 | 입력 내용이 서버 기록으로 남을 수 있음 | 학생 개인정보 입력을 차단하고 수집을 최소화 |
| 교사 통제 | 약함 | 주제, 목표, 힌트 강도, 페르소나, 공유 상태를 교사가 설정 |
| 관리자 운영 | 별도 운영 흐름 필요 | 관리자 승인, 계정 관리, 모델 설정, 사용량 확인 제공 |

## 5. 현재 구현된 주요 기능

### 교사 가입과 승인

- 이메일 로그인과 Google 로그인을 지원합니다.
- 회원가입은 이름, 이메일, 비밀번호 확인, 학교 검색과 선택을 거쳐 `가입 요청`으로 일원화했습니다.
- 학교는 직접 문자열만 입력하는 방식이 아니라 NEIS 학교 검색 결과에서 선택합니다.
- 교사 계정은 관리자 승인 후 사용할 수 있습니다.
- 로그인 입력 후 Enter로 로그인할 수 있습니다.
- 로그인 처리 중에는 `로그인 정보를 확인하고 있습니다. 잠시만 기다려 주세요.` 안내를 표시합니다.

### 관리자 계정

- `KKOKKOMU_ADMIN_EMAILS`에 등록된 이메일은 관리자 계정으로 자동 승격됩니다.
- 이미 일반 승인 교사 프로필로 만들어진 관리자 이메일도 로그인 시 관리자 권한으로 승격됩니다.
- 관리자는 관리자 대시보드와 교사용 챗봇 만들기 화면을 함께 볼 수 있습니다.
- 관리자 전용 API는 관리자 프로필일 때만 호출해 일반 교사 로그인에서 불필요한 403 콘솔 오류가 나지 않도록 했습니다.
- 관리자는 교사 승인, 거절, 비활성화, 비밀번호 초기화 메일 발송, 챗봇 비활성화, AI 모델 설정, 관리자 작업 로그 확인을 할 수 있습니다.

### 계정 관리

- 상단 nav 프로필 아이콘 왼쪽에 로그아웃 버튼을 제공합니다.
- 프로필 패널에서 나의 정보, 이름, 이메일, 학교 정보를 확인할 수 있습니다.
- 비밀번호 변경은 버튼을 눌렀을 때만 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인 입력칸이 열립니다.
- 비밀번호를 잊었을 때는 `greenguyhh@gmail.com`으로 문의하도록 안내합니다.
- 회원탈퇴 버튼은 위험 작업으로 분리하고, 계정 비활성화 흐름으로 처리합니다.

### 챗봇 생성

- 교사는 챗봇 이름, 학교급, 학년군, 과목, 수업 주제, 학습 목표, 힌트 강도, 페르소나를 설정합니다.
- 챗봇 이름 예시는 `국어 9품사 이해`처럼 특정 학년을 반드시 써야 할 것처럼 보이지 않게 정리했습니다.
- 수업 주제와 목표를 바탕으로 교육과정 성취기준을 추천합니다.
- 추천 성취기준은 여러 개 선택할 수 있고, 선택하지 않으면 가장 관련 높은 기준을 자동 반영합니다.
- 생성한 챗봇은 공유 링크를 켜서 학생에게 전달할 수 있습니다.
- 챗봇 삭제 시 교사 목록에서 즉시 제거되고 기존 학생 공유 링크는 사용할 수 없게 됩니다.

### 학생 화면

- 학생은 회원가입 없이 공유 링크로만 접속합니다.
- 학생 화면에는 교사/관리자 기능이 보이지 않고, 질문과 대화에 집중하는 화면만 표시됩니다.
- 학생은 이름, 학번, 연락처, 주소, 이메일 등 개인정보를 입력하지 않도록 안내받습니다.
- 개인정보로 보이는 입력은 AI 전송 전에 차단합니다.
- 대화 기록은 기본적으로 학생 브라우저에만 저장되며, 학생이 직접 PDF 또는 TXT로 내려받을 수 있습니다.

### 보안과 운영

- Firebase ID 토큰을 서버에서 검증합니다.
- Firebase Admin Auth 번들 문제를 피하기 위해 Identity Toolkit REST 검증 흐름을 사용합니다.
- API 키와 서버 전용 설정은 클라이언트 코드에 노출하지 않습니다.
- Vercel 배포 smoke 테스트에서 SPA, 개인정보처리방침, API health, 인증 없는 교사 API 차단, 보안 헤더, CORS preflight 차단을 확인합니다.
- 개인정보처리방침에는 개인정보 보호책임자 및 문의 이메일 `greenguyhh@gmail.com`을 명시했습니다.

## 6. 발표 시연 흐름

### 장면 1: 교사 로그인

- 이메일 또는 Google 로그인
- 로그인 중 안내 문구 확인
- 승인된 교사는 챗봇 만들기 화면으로 이동
- 관리자 이메일은 관리자 대시보드와 교사 화면을 함께 확인

보여줄 메시지:

> 계정은 단순 로그인으로 끝나는 것이 아니라, 학교 확인과 관리자 승인 이후 수업 도구로 사용할 수 있습니다.

### 장면 2: 챗봇 만들기

- 챗봇 이름 입력
- 학교급, 학년군, 과목 입력
- 수업 주제와 학습 목표 입력
- 관련 성취기준 추천 확인
- 힌트 강도와 페르소나 설정
- 챗봇 생성

보여줄 메시지:

> 교사가 복잡한 프롬프트를 직접 만들지 않아도 됩니다. 수업 주제와 목표를 넣으면 교육과정 기반 챗봇을 만들 수 있습니다.

### 장면 3: 성취기준 선택

- 기본 추천 성취기준 확인
- 더 보기로 추가 성취기준 확인
- 필요한 기준을 하나 이상 선택
- 선택하지 않아도 가장 관련 높은 기준이 자동 반영됨

보여줄 메시지:

> 추천은 강제가 아닙니다. 교사가 직접 고를 수 있고, 고르지 않아도 가장 관련 높은 기준이 자동으로 반영됩니다.

### 장면 4: 학생 공유 링크

- 공유 링크 생성
- 학생용 링크로 접속
- 교사/관리자 기능이 없는 학생 화면 확인

보여줄 메시지:

> 학생에게는 필요한 화면만 보여 줍니다. 수업 중에는 링크를 열고 질문을 이어 가는 경험에 집중하게 합니다.

### 장면 5: 정답 요구 차단

학생:

> 답만 알려 주세요.

챗봇:

> 답만 바로 알려주기보다 같이 하나씩 찾아보면 좋겠어요. 먼저 어떤 부분이 헷갈렸나요?

보여줄 메시지:

> 이 챗봇은 정답 자동 판매기가 아닙니다. 학생이 생각을 시작하게 만드는 수업 도구입니다.

### 장면 6: 개인정보 차단

- 이름, 전화번호, 주소처럼 개인정보로 보이는 입력을 시도
- AI 전송 전에 다시 작성하도록 안내

보여줄 메시지:

> 학생 개인정보를 기본적으로 받지 않는 구조입니다. 입력 단계에서도 개인정보 의심 내용을 먼저 차단합니다.

### 장면 7: 관리자 운영

- 가입 요청 승인 또는 거절
- 교사 계정 비활성화
- 비밀번호 초기화 메일 발송
- 사용량과 AI 모델 설정 확인
- 관리자 작업 로그 확인

보여줄 메시지:

> 학교 현장에서 쓰려면 챗봇 기능만으로는 부족합니다. 계정 승인, 권한, 사용량, 로그까지 운영 흐름이 필요합니다.

## 7. 예상 질문과 답변

### Q1. 그냥 ChatGPT를 쓰면 안 되나요?

일반 챗봇은 답을 잘 주는 방향으로 최적화되어 있습니다. 꼬꼬무AI는 반대로 정답을 바로 주지 않도록 설계했습니다. 교사가 정한 수업 범위와 교육과정 기준 안에서 학생이 스스로 생각하도록 돕는 점이 다릅니다.

### Q2. 교육과정 원문만 보고 답하나요?

아닙니다. 교육과정은 기준선과 경계선입니다. 실제 대화에서는 학생 수준에 맞는 예시, 비유, 힌트, 질문을 사용합니다. 다만 교사가 정한 학교급, 과목, 주제, 성취기준을 벗어나지 않도록 제한합니다.

### Q3. 학생 개인정보는 어떻게 처리하나요?

학생은 가입하지 않고 이름과 학번도 받지 않습니다. 학생 대화도 기본적으로 서버에 저장하지 않습니다. 대화 기록은 학생 기기 브라우저에만 저장되며, 필요하면 PDF나 TXT로 내려받아 별도 플랫폼에 제출할 수 있습니다.

### Q4. 학생이 숙제 답만 요구하면요?

최종 답을 직접 제공하지 않습니다. 어디에서 막혔는지 확인하고, 다음 한 단계만 생각하도록 질문합니다.

### Q5. 교사가 성취기준을 몰라도 쓸 수 있나요?

쓸 수 있습니다. 교사가 수업 주제를 입력하면 관련 교육과정과 성취기준을 추천합니다. 추천은 참고용이고 교사가 직접 선택할 수 있습니다.

### Q6. 관리자는 어떻게 만들어지나요?

운영 환경의 `KKOKKOMU_ADMIN_EMAILS`에 관리자 이메일을 등록합니다. 해당 이메일로 로그인하면 서버가 관리자 프로필을 자동 생성하거나 기존 교사 프로필을 관리자 권한으로 승격합니다.

### Q7. 비밀번호를 잊은 교사는 어떻게 하나요?

관리자가 교사 계정에 대해 비밀번호 초기화 메일을 보낼 수 있습니다. 사용자는 안내된 문의 이메일 `greenguyhh@gmail.com`으로 도움을 요청할 수 있습니다.

## 8. 마무리 문장

꼬꼬무AI는 AI를 더 빠른 정답 생성기로 쓰려는 시도가 아닙니다.

AI를 교사의 수업 의도 안에 묶고, 학생이 스스로 생각하는 시간을 지켜 주려는 도구입니다.

그래서 이 챗봇의 목표는 답을 대신 말하는 것이 아니라, 학생이 자기 생각으로 답에 가까워지도록 돕는 것입니다.
